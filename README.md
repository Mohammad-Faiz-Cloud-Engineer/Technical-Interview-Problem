# Technical Interview Problem

A comprehensive collection of algorithmic problems organized by company interview patterns. This repository serves as a structured resource for software engineers preparing for technical interviews at leading technology companies.

## About This Project

Maintained by Mohammad Faiz, this repository aggregates and organizes coding problems based on actual interview patterns observed across the industry. The data is systematically updated to reflect current hiring practices as of June 2025.

## Repository Structure

Each company directory contains five distinct problem sets, organized by temporal relevance:

**File Organization:**
- `1. Thirty Days.csv` - Recent problems (last 30 days)
- `2. Three Months.csv` - Quarterly problem set (last 90 days)
- `3. Six Months.csv` - Semi-annual problem set (last 180 days)
- `4. More Than Six Months.csv` - Historical problems (beyond 6 months)
- `5. All.csv` - Complete aggregated problem set

This temporal categorization allows candidates to prioritize recent interview patterns while maintaining access to comprehensive historical data.

## Methodology

The problem sets are compiled through systematic analysis of interview reports, online coding platforms, and publicly available interview experiences. Each entry is verified and categorized to ensure accuracy and relevance.

## Usage Guidelines

**For Interview Preparation:**
1. Identify your target company within the repository
2. Begin with the most recent problem set (30-day file) to understand current interview trends
3. Progress through older problem sets to build comprehensive coverage
4. Reference the complete problem set for holistic preparation

**Strategic Approach:**
- Recent problems indicate current technical focus areas
- Historical problems provide pattern recognition and foundational practice
- Cross-reference multiple companies to identify common algorithmic themes
